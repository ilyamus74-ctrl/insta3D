# Актуализация ТЗ (web-клиент сейчас, backend позже)

Дата фиксации: 2026-04-20

## 0) Платформа (зафиксировано)
- Клиент: Web App (адаптивный интерфейс для телефонов, планшетов и ПК).
- Сервер обработки: Linux (backend и pipeline обработки будут добавлены после проверки работы приложения с Insta360).
- Текущий фокус: фронтенд-приложение + интеграционный слой с камерой (через mock/адаптер), без полной серверной реализации.

## 1) MVP-сценарии (3–5 ключевых)

Ниже 5 сценариев, которые закрывают первый запуск и тесты в поле:

1. **Создание проекта/сессии съемки**
   - Оператор создает локальную сессию: название объекта, адрес, комментарий.
   - Все данные сохраняются локально в браузере (offline-first).

2. **Подключение к Insta360 и проверка статуса**
   - Экран подключения к камере.
   - Статус: подключено/не подключено, модель, заряд, свободная память (если доступно через API/bridge).

3. **Съемка точек и журнал точек**
   - Кнопка «Снять точку».
   - После съемки добавляется карточка точки: имя, время, статус, превью (если есть).
   - Возможность переименования и удаления точки из черновика.

4. **Черновик тура (порядок обхода)**
   - Список снятых точек с drag-and-drop сортировкой.
   - Простая связность: следующая/предыдущая точка (без production viewer).

5. **Очередь отправки и статусы синхронизации (mock)**
   - Подготовка пакета выгрузки.
   - Статусы: «в очереди», «отправка», «успешно», «ошибка».
   - На этом этапе backend можно заменить mock API.

---

Сразу важное ограничение: у Insta360 есть официальный Android SDK для подключения, управления съемкой, получения параметров камеры, загрузки файлов и экспорта, но доступ к SDK завязан на их developer-программу. Также у них есть OSC/HTTP-протокол для управления камерами по Wi-Fi. Еще нюанс: при подключении телефона к камере по Wi-Fi интернет у других приложений на Android может пропадать, а в фирменном приложении это решается через Proxy Mode. 


#1. Цель проекта

Разработать Android-приложение MVP для операторов съемки недвижимости, которое:

    подключается к камере Insta360 по Wi-Fi

    управляет съемкой 360-панорам

    показывает превью отснятых точек

    позволяет собрать черновик тура прямо на телефоне

    не выгружает оригиналы по мобильной сети по умолчанию

    откладывает полную выгрузку до появления нормального Wi-Fi

    отправляет материалы на сервер для финальной сборки тура

#2. Границы MVP
Входит в MVP

    авторизация оператора

    список объектов/заявок

    подключение к Insta360

    съемка точек

    просмотр списка отснятых точек

    черновик маршрута тура

    локальное хранение metadata и preview

    очередь загрузки

    выгрузка оригиналов только по Wi-Fi

    отправка на backend

    получение статуса обработки

Не входит в MVP

    финальная сборка production-тура на телефоне

    тяжелый stitching/export full-quality на телефоне

    автоматическое распознавание комнат

    автопостроение идеального маршрута без участия человека

    встроенный клиентский viewer production-уровня

    биллинг и кабинет маклера внутри приложения

#3. Основной сценарий работы
Сценарий оператора

    Оператор входит в приложение.

    Открывает назначенный объект.

    Подключается к Insta360 по Wi-Fi.

    Видит статус камеры:

        модель

        заряд

        свободное место

        режим съемки

    Нажимает “Снять точку”.

    После съемки приложение:

        получает preview/thumbnail

        сохраняет локальную карточку точки

        предлагает назвать точку: Прихожая 1, Кухня, Спальня 2

    Оператор снимает весь объект.

    На экране “Черновик тура” видит:

        список точек

        превью

        порядок обхода

        простые связи между точками

    После завершения выбирает:

        Сохранить локально

        Ждать Wi-Fi для выгрузки

        Выгрузить сейчас только если доступен нормальный Wi-Fi

    После выгрузки backend собирает финальный тур.

    Приложение показывает статус:

    загружено

    обрабатывается

    готово

    ошибка

#4.3 Camera Integration Layer

Два режима интеграции:
Вариант A — через Insta360 Android SDK

Предпочтительный путь, если получишь доступ. Официально SDK умеет подключение, параметры камеры, управление съемкой, скачивание файлов и экспорт.
Вариант B — через Insta360 OSC/HTTP

Резервный путь. Insta360 публикует OSC-команды для управления камерами через HTTP по Wi-Fi AP камеры.

#5.2 Работа с объектами

Приложение должно:

    получать список назначенных объектов

    показывать:

        ID объекта

        адрес

        клиент/маклер

        статус

        дата/время заявки

    позволять создать локальную съемочную сессию

#5.3 Подключение к Insta360

Приложение должно:

    показывать статус камеры

    инициировать подключение по Wi-Fi

    определять модель камеры

    показывать ошибки подключения

    проверять совместимость модели

Поддерживаемые модели надо зафиксировать отдельно. На публичной странице Android SDK перечислены как минимум X5, X4, X3, ONE X2, ONE X, ONE RS, ONE RS 1-Inch.

#5.4 Управление съемкой

Приложение должно:

    запускать съемку фото/панорамы

    получать подтверждение завершения съемки

    отображать прогресс

    не допускать повторной команды до завершения предыдущей
    Это особенно важно, потому что в OSC Insta360 рекомендует не отправлять следующую команду до ответа на предыдущую.

#5.5 Локальная карточка точки

После каждой съемки приложение сохраняет:

    point_id

    object_id

    session_id

    local_timestamp

    camera_file_id или путь к файлу

    preview_path

    title

    room_type

    sequence_number

    yaw/pitch/roll если доступны

    notes

    upload_status

#5.6 Черновик тура

Приложение должно:

    показывать список отснятых точек

    показывать превью

    позволять менять порядок

    позволять вручную задавать связи:

        точка A -> точка B

    поддерживать стартовую точку

    поддерживать пометку “комната / коридор / санузел / балкон / вход”

Черновик — это не production-тур, а съемочная схема.
#5.7 Очередь загрузки

Приложение должно:

    формировать очередь оригиналов

    загружать previews отдельно от originals

    уметь режимы:

        только preview

        полная выгрузка только по Wi-Fi

        ручная полная выгрузка

    возобновлять загрузку после разрыва

    показывать прогресс по каждому объекту

#5.8 Ограничения по сети

По умолчанию:

    не выгружать оригиналы по мобильной сети

    предупреждать, если телефон подключен к Wi-Fi камеры и интернет недоступен

    поддерживать отложенную синхронизацию

Это особенно актуально, потому что Insta360 прямо пишет, что при Wi-Fi-подключении к камере другие приложения на Android могут терять доступ в интернет, а в фирменном приложении это обходится Proxy Mode. В своей app это надо либо поддержать аналогично, либо явно учитывать в UX. 


#6. Нефункциональные требования
#6.1 Производительность

    приложение не должно падать на сессии до 50 точек

    список точек должен открываться менее чем за 2 секунды

    превью должны кешироваться локально

    все тяжелые операции выполнять в фоне

#6.2 Хранение

Локально на телефоне хранить:

    metadata

    thumbnails/previews

    временно originals до выгрузки

Приложение должно показывать:

    сколько занимает текущая съемка

    сколько осталось до выгрузки

    можно ли очистить локальные originals после успешной загрузки

#6.3 Надежность

    при убийстве приложения сессия не теряется

    очередь загрузки восстанавливается

    все действия логируются

    все ошибки соединения с камерой и сервером логируются

#6.4 Безопасность

    токены хранить в EncryptedSharedPreferences / Android Keystore

    локальные служебные данные шифровать по возможности

    медиа отправлять только по HTTPS

    поддержать удаление локальных originals после подтвержденной загрузки

#11. Логика работы с сетью
Состояния сети

    CAMERA_WIFI_NO_INTERNET

    CAMERA_WIFI_WITH_PROXY

    NORMAL_WIFI

    MOBILE_DATA

    OFFLINE

Правила

    если CAMERA_WIFI_NO_INTERNET:

        съемка разрешена

        загрузка на сервер запрещена

    если NORMAL_WIFI:

        разрешена полная выгрузка

    если MOBILE_DATA:

        только previews или только при ручном подтверждении

    если OFFLINE:

        только локальная работа

#12. Ограничения и риски
#12.1 Риск по SDK

Доступ к Insta360 SDK может потребовать одобрения/аккаунта разработчика. Это надо проверить и заложить в проект сразу.
#12.2 Риск по моделям камер

Надо фиксировать список поддерживаемых моделей MVP, а не пытаться “поддержать все”.
#12.3 Риск по интернету

Когда телефон сидит на Wi-Fi камеры, интернет может быть недоступен. Это не баг твоей логики, а особенность сценария.
#12.4 Риск по ресурсу телефона

Нельзя закладываться на тяжелую финальную обработку на телефоне. Телефон только:

    снимает

    показывает preview

    ведет черновик

    выгружает позже

#14. Критерии приемки MVP

MVP считается готовым, если:

    оператор может открыть объект и начать съемку

    приложение подключается к Insta360

    можно снять не менее 20 точек подряд без потери сессии

    после каждой точки появляется превью

    можно собрать черновой маршрут

    originals не грузятся по мобильной сети по умолчанию

    по Wi-Fi идет полная выгрузка с возобновлением

    backend получает полный пакет данных

    после успешной выгрузки локальные originals можно очистить

    приложение восстанавливает незавершенную сессию после перезапуска
